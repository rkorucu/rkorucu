Agentic Claim Decision Engine

AI-powered insurance claim analysis system that combines deterministic fraud/coverage checks with optional LLM explanation and self-checking.

View Repository

Problem

Insurance claim review is either fully manual (slow, inconsistent) or based on rigid rule engines that can't explain their decisions. This system automates claim adjudication using structured tools for fraud detection and coverage validation, with an optional LLM layer that generates human-readable explanations and self-checks for consistency.

How It Works

  • User submits a claim via the React dashboard. Spring Boot gateway validates the payload and forwards it to the FastAPI agent service.
  • coverage_check validates policy constraints — active status, coverage type, deductible, and claim limits.
  • fraud_signal_scoring scores fraud indicators — prior claims count, missing police reports, attachment gaps.
  • risk_score_aggregation combines signals into a composite risk score and maps it to APPROVE / REVIEW / REJECT.
  • If an OpenAI key is configured, an LLM generates a plain-language explanation and a reflection step checks for contradictions between the score, decision, and explanation.
  • Without an API key, the system returns a fully valid deterministic-only decision — no AI dependency required.

SkillBridge — AI Expert Matching Platform

An expert marketplace that uses semantic embeddings and LLM-powered matching to connect clients with tutors, consultants, and coaches.

View Repository

Problem

Traditional expert marketplaces rely on keyword search and manual filtering. A client looking for "help optimizing database queries for a multi-tenant SaaS app" won't match an expert tagged "SQL" and "PostgreSQL" — even though the expertise fits. This platform uses semantic understanding to match based on actual expertise context, not just keywords.

How It Works

  • Expert profiles are ingested with structured metadata (skills, domains, experience level) and converted into vector embeddings.
  • Client requests are processed through an LLM to extract intent and required expertise, producing a structured matching query.
  • Embedding similarity scores rank experts by relevance — capturing contextual alignment beyond keyword overlap.
  • Matched pairs enter a session lifecycle: scheduling, active session, and post-session AI-generated summaries with key takeaways and action items.
  • Falls back to keyword-based search when embedding services are unavailable.

Aegis Orchestrator — Multi-Agent AI Platform

A graph-based multi-agent orchestration platform where specialized AI agents collaborate through a structured execution graph with validated outputs and bounded reflection.

View Repository

Problem

Single-call LLM applications can't handle tasks that need multi-perspective analysis, structured outputs, and self-correction. Aegis decomposes complex queries into a directed graph of agents — Planner, Research, Risk, Critic, Synthesizer — each with a defined role, validated output schema, and access to deterministic tools.

How It Works

  • Planner breaks the user query into a structured execution plan with numbered steps.
  • Research gathers data using deterministic tools (market analysis, financial projections, risk scoring) and LLM reasoning.
  • Risk evaluates potential risks, assigns severity levels, and proposes mitigations.
  • Critic reviews all outputs for quality and consistency — emits APPROVE or REVISE. Reflection is hard-capped at 2 loops.
  • Synthesizer produces the final structured response with key points, recommendation, and confidence level.
  • All agents share a single typed state object. Every output is JSON-schema validated before entering state. SSE streaming shows real-time execution progress on the frontend.

System Design Philosophy

State-Driven Design

Every system I build has an explicit state model. Components transition between well-defined states with validated inputs and outputs at each boundary. If the system can't tell you exactly what state it's in and why — the design is incomplete.

Deterministic + Generative Hybrid

LLMs are powerful, but unreliable when unconstrained. I design systems where deterministic tools handle the critical path — scoring, validation, decision logic — and generative AI is sandboxed to explanation, summarization, and reflection. The system works without AI. AI makes it better.

Failure Mode Thinking

Every integration point, every external service call, every state transition has a defined failure mode. Graceful degradation is not an afterthought — it's a design constraint. Systems must produce correct, if reduced, output when dependencies fail.

Hallucination Risk Control

In AI-integrated systems, hallucination is a reliability failure, not a feature limitation. I mitigate it through structured output schemas, deterministic tool grounding, post-generation validation, reflection-based consistency checks, and strict separation between generative and authoritative data paths.

Observability as a First-Class Concern

If I can't observe it, I can't trust it. Every system I build includes structured logging, health checks, execution tracing, and decision audit trails. In AI systems especially, observability is what separates a prototype from a production platform.
